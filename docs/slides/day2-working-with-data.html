The Pipe Operator

Introducing |>

The pipe passes the result to the next function

# Without pipe
head(corona)

# With pipe
corona |> head()

Read |> as “and then”

Why Use the Pipe?

Makes code more readable:

# Nested (hard to read)
head(filter(select(corona, author_name, view_count), view_count > 100000))

# Piped (easy to read)
corona |>
  select(author_name, view_count) |>
  filter(view_count > 100000) |>
  head()

The Pipe Pattern

data |>
  do_something() |>
  do_another_thing() |>
  do_more()

Keyboard shortcut: Ctrl+Shift+M (Windows/Linux) or Cmd+Shift+M (Mac)

Combining Conditions: AND

Both conditions must be true

# Use & or comma
corona |> filter(view_count > 100000 & duration < 60)

corona |> filter(view_count > 100000, duration < 60)

Combining Conditions: OR

At least one condition must be true

# Use |
corona |> filter(view_count > 1000000 | like_count > 100000)

Multiple Values: %in%

Check if value is one of several options:

Categorizing Data

case_when(): Create Categories

corona <- corona |>
  mutate(
    duration_category = case_when(
      duration < 30 ~ "short",
      duration >= 30 & duration <= 60 ~ "medium",
      duration > 60 ~ "long"
    )
  )

case_when() Syntax

case_when(
  condition1 ~ "result1",
  condition2 ~ "result2",
  condition3 ~ "result3"
)
  • Conditions are tested in order
  • First TRUE condition wins
  • Use TRUE ~ "default" for a catch-all

The Pipe Deep Dive

Without the Pipe

Nested functions (hard to read!):

head(filter(select(corona, author_name, view_count),
            view_count > 100000), 10)

You read from inside out: select → filter → head

With the Pipe

Step by step (easy to read!):

corona |>
  select(author_name, view_count) |>
  filter(view_count > 100000) |>
  head(10)

You read top to bottom: select → filter → head

Building Complex Transformations

corona |>
  filter(view_count > 50000) |>
  mutate(eng_per_1k = engagement_total / (view_count / 1000)) |>
  select(author_name, view_count, eng_per_1k) |>
  arrange(desc(eng_per_1k)) |>
  head(10)

Important: ungroup()

If you continue working with grouped data, ungroup first:

corona_with_daily <- corona |>
  group_by(create_date) |>
  mutate(daily_avg = mean(view_count)) |>
  ungroup()

Otherwise grouping persists and may cause unexpected behavior!

Verb Action Works on

The Workflow

data |>
  filter(...) |>      # subset rows
  select(...) |>      # choose columns
  mutate(...) |>      # create variables
  group_by(...) |>    # define groups
  summarize(...) |>   # calculate stats
  arrange(...)        # sort results

The Grammar of Graphics

ggplot2: Building Plots in Layers

Every plot has:

  1. DATA — What dataset?
  2. AESTHETICS — What variables map to x, y, color, size?
  3. GEOMETRIES — What shapes represent the data?
  4. LAYERS — Labels, themes, scales…

The Basic Template

ggplot(data = <DATA>, aes(x = <X_VAR>, y = <Y_VAR>)) +
  geom_<TYPE>()

Note: We use + to add layers (not |>)

Your First Plot

ggplot(data = corona, aes(x = view_count, y = like_count)) +
  geom_point()
  • Data: corona
  • Aesthetics: x = view_count, y = like_count
  • Geometry: points

Compact Syntax

You can omit data = and x =, y =:

ggplot(corona, aes(view_count, like_count)) +
  geom_point()

Histograms

geom_histogram(): One Variable Distribution

ggplot(corona, aes(x = view_count)) +
  geom_histogram()

Shows how values are distributed

Controlling Bins

ggplot(corona, aes(x = view_count)) +
  geom_histogram(bins = 50)

More bins = more detail (but can be noisy)

Problem: Skewed Data

Most videos have few views, some have millions

Solution: Log scale

ggplot(corona, aes(x = view_count)) +
  geom_histogram(bins = 50) +
  scale_x_log10()

Polished Histogram

ggplot(corona, aes(x = view_count)) +
  geom_histogram(bins = 50, fill = "steelblue", color = "white") +
  scale_x_log10(labels = label_comma()) +
  labs(
    title = "Distribution of Video Views",
    x = "Views (log scale)",
    y = "Number of Videos"
  )

Understanding Box Plots

  • Box: 25th to 75th percentile (middle 50%)
  • Line: Median (50th percentile)
  • Whiskers: 1.5 × IQR from box
  • Points: Outliers beyond whiskers

words <- corona |>

Step 2: Remove Stopwords

words_clean <- words |>
  anti_join(italian_stopwords, by = "word")

nrow(words_clean)

anti_join() keeps only rows that don’t match

Step 3: Additional Cleaning

words_clean <- words_clean |>
  filter(!str_detect(word, "^[0-9]+$")) |>    # Remove numbers

  filter(!word %in% c("https", "http", "www", "com", "tiktok"))
