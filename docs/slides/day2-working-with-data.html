Data Cleaning Basics

Real Data Is Messy

Remember from Day 1: our dataset was collected via keyword and hashtag search.

  • Some values may be missing (NA)
  • There could be duplicates
  • Some videos may be false positives

First step: understand what needs cleaning

Checking for Missing Values

# Count NAs in each column
colSums(is.na(corona))
# Which columns have missing values?
corona |>
  summarize(
    missing_description = sum(is.na(description)),
    missing_hashtags = sum(is.na(hashtags))
  )

What Is NA?

NA means “Not Available” — the value is missing

x <- c(10, NA, 30)
mean(x)          # Returns NA!
mean(x, na.rm = TRUE)  # Ignores NA → returns 20

na.rm = TRUE is your friend for calculations with missing data

Checking for Duplicates

Our dataset was already de-duplicated, but it’s good practice to verify:

# Are there duplicate video IDs?
corona |>
  count(video_id, sort = TRUE) |>
  filter(n > 1)

Note: videos with the same description are OK — different creators can post similar content. Always de-duplicate by unique identifier (video_id), not by content.

# If needed:
corona <- corona |> distinct(video_id, .keep_all = TRUE)

Filtering Out Bad Data

# Remove videos with no description AND no hashtags
corona_clean <- corona |>
  filter(!is.na(description) | !is.na(hashtags))

# Remove very short videos (< 1 second — likely errors)
corona_clean <- corona_clean |>
  filter(duration >= 1)

Note: a missing description is fine if the video has hashtags — we still have useful text data.

Tip: always keep the original data and create a new variable for cleaned data

What About False Positives?

The only way to completely eliminate false positives is to manually check every post.

This can be partially automated using an LLM (Large Language Model) in the loop to classify each post as relevant or not.

This falls outside the scope of this course, but it’s important to keep in mind:

Your dataset may contain noise — and that’s OK for learning.

Cleaning Workflow

Import → Explore → Understand → Clean → Analyze
  1. Explore the data first (glimpse, summary)
  2. Identify problems (NAs, duplicates, outliers)
  3. Decide what to keep and what to remove
  4. Document your choices — no “right” answer!

The Pipe Operator

Introducing |>

The pipe passes the result to the next function

# Without pipe
head(corona)

# With pipe
corona |> head()

Read |> as “and then”

Why Use the Pipe?

Makes code more readable:

# Nested (hard to read)
head(filter(select(corona, author_name, view_count), view_count > 100000))

# Piped (easy to read)
corona |>
  select(author_name, view_count) |>
  filter(view_count > 100000) |>
  head()

The Pipe Pattern

data |>
  do_something() |>
  do_another_thing() |>
  do_more()

Keyboard shortcut: Ctrl+Shift+M (Windows/Linux) or Cmd+Shift+M (Mac)

Select a Range

corona |> select(video_id:duration)

Selects all columns from video_id to duration

Remove Columns

Use - to exclude:

corona |> select(-music_id, -region_code)

Keeps everything except music_id and region_code

Select by Pattern

# Columns ending with "_count"
corona |> select(ends_with("_count"))

# Columns starting with "create"
corona |> select(starts_with("create"))

# Columns containing "count"
corona |> select(contains("count"))

Select Helpers Summary

Helper Matches
ends_with("x") Columns ending with “x”
starts_with("x") Columns starting with “x”
contains("x") Columns containing “x”
everything() All remaining columns

corona |> select(ends_with("_count"), -view_count)

Filtering Rows

filter(): Choose Rows

Keep rows where a condition is TRUE

# Videos with more than 1 million views
corona |> filter(view_count > 1000000)

Comparison Operators

Operator Meaning
== Equal to
!= Not equal to
> Greater than
< Less than
>= Greater than or equal
<= Less than or equal

filter() Examples

# Videos from a specific date
corona |> filter(create_date == "2026-01-15")

# Videos shorter than 30 seconds
corona |> filter(duration < 30)

# Videos longer than 60 seconds
corona |> filter(duration > 60)

Combining Conditions: AND

Both conditions must be true

# Use & or comma
corona |> filter(view_count > 100000 & duration < 60)

corona |> filter(view_count > 100000, duration < 60)

Combining Conditions: OR

At least one condition must be true

# Use |
corona |> filter(view_count > 1000000 | like_count > 100000)

Multiple Values: %in%

Check if value is one of several options:

corona |> filter(create_date %in% c("2026-01-01",
                                     "2026-01-02",
                                     "2026-01-03"))

Counting Results

How many rows match your filter?

corona |> filter(view_count > 1000000) |> nrow()
[1] 42

Try It: filter()

Find videos with:

  • More than 10,000 likes AND
  • More than 100 comments

How many are there?

corona |>
  filter(like_count > 10000, comment_count > 100) |>
  nrow()

Combining Operations

select() + filter()

corona |>
  filter(view_count > 500000) |>
  select(author_name, description, view_count, like_count)

Read as: Take corona, then filter to high-view videos, then select key columns

Example: First Week of January

corona |>
  filter(create_date >= "2026-01-01",
         create_date <= "2026-01-07") |>
  select(video_id, create_date, ends_with("_count"))

The Power of Chaining

corona |>
  filter(view_count > 100000) |>
  filter(duration < 60) |>
  select(author_name, view_count, duration) |>
  head(10)

Each step transforms the data for the next step
